# M10 アレンジとテクスチャ

**今回の Strudel:** `.mask("<1 0>")` / `.every(n, x => ...)` / 信号でのオートメーション
**今回の音楽理論:** 密度 / テクスチャ / レイヤーの足し引きによる展開

## テクスチャと密度

同じループでも、鳴っているレイヤーの数(密度)で印象が変わります。すべてのレイヤーがずっと鳴り続けると単調になりますが、レイヤーを足したり抜いたりすることで聴き手を飽きさせない展開を作れます。

## mask で抜く

`.mask("<1 1 1 0>")` はサイクルごとに 1=鳴らす / 0=止める という指示です。4 サイクルめにハットを抜くと「溜め」ができ、次のサイクルで再び入ってくる瞬間に盛り上がりを作れます。

```js
// M10 アレンジとテクスチャ(レイヤーの足し引き)
setcpm(124 / 4)
stack(
  s("bd*4"),
  s("hh*8").gain(0.6).mask("<1 1 1 0>"),          // 4 サイクルめだけハットを抜く
  s("rim*16").gain(0.4).every(4, x => x.fast(2)), // 4 サイクルごとに密度を上げる
)
```

## every で変化

`.every(4, x => x.fast(2))` は 4 サイクルごとに関数を適用します。ここでは密度を 2 倍にしていますが、`x.rev()`(逆再生)や `x.slow(2)`(ゆっくり)など他の変形も使えます。変化のタイミングを mask と合わせると、より音楽的な展開になります。

## オートメーション

M5 の `lpf(sine.range(...).slow(...))` のように、信号でパラメータをゆっくり動かすのもテクスチャ作りの手法です。フィルターの開閉をサイクル単位で自動的に変化させると、機械的な繰り返しに有機的な動きが加わります。
